1. Introduction

NeoLauncher is designed to provide launcher, search, widget, and personalization features while keeping data collection as limited as possible.

Most data used by NeoLauncher remains on your device and is processed locally. Some information may be sent to external services only when needed to provide certain optional features, such as crash reporting, weather, geocoding, or web search.

NeoLauncher does not sell your personal data and does not share personal data with third parties for advertising purposes.

2. Information We Collect

The app may access, process, or store the following categories of information:

  • Crash diagnostics collected through Firebase Crashlytics, including crash reports, stack traces, technical error information, app version, and limited diagnostic app information such as launcher style.
  • Installed app metadata, such as app labels, icons, and component information, to display and organize the launcher and app drawer.
  • Contacts, if you grant permission, to support contact search functionality.
  • Calendar information, if you grant permission, to show upcoming events and calendar-related launcher features.
  • Location data, if you grant permission, to provide weather-related functionality.
  • Notification-related data, such as app package names and unread notification counts, if you enable notification access for unread badge features.
  • Battery state, such as battery level and charging state, for battery ring and related overlay features.
  • Camera availability state, only to optionally hide the battery ring while the front camera is in use.
  • User-entered search queries, when you use search suggestions or web search features.
  • Media session information, such as media title, artist, album art, and app package, when media-related launcher features are used.
  • App settings, layout preferences, workspace data, dock configuration, and personalization choices stored locally on your device.
Most information used by NeoLauncher stays on your device and is processed locally. Optional features only access the data needed for that feature to work.

3. How We Use Information

NeoLauncher uses information only to provide, maintain, and improve app functionality. This includes:

  • Displaying installed apps, home screen items, folders, and app drawer content
  • Providing contact search and calendar event features when permission is granted
  • Providing weather information and location-based weather display when permission is granted
  • Supporting search suggestions and web search actions
  • Displaying unread badge counts for supported apps
  • Displaying battery ring and edge lighting overlays
  • Showing media-related launcher widgets or media information where applicable
  • Saving launcher settings, theme choices, layout data, and backup/restore information
  • Monitoring crashes and technical issues to improve stability and fix bugs

4. Data Sharing and Third Parties

NeoLauncher does not sell your personal data. NeoLauncher also does not share personal data with third parties for advertising purposes.

Some information may be sent to external services only when required to provide optional features:

  • Firebase Crashlytics for crash reporting and technical diagnostics
  • Open-Meteo for weather data requests
  • Nominatim / OpenStreetMap for geocoding or location-related lookups
  • Search providers such as Google, Bing, DuckDuckGo, Yahoo, or Brave when you use web search or search suggestion features

Contacts, calendar data, notification badge counts, media information, battery state, and most personalization data are processed locally and are not transmitted by those features themselves.

5. No Advertising or Tracking

NeoLauncher does not display advertisements and does not include advertising SDKs.

NeoLauncher does not track users for marketing, behavioral advertising, or profiling purposes.

The only third-party service currently used in the app is Firebase Crashlytics, which is used solely for crash reporting and stability monitoring.

6. Data Storage and Retention

NeoLauncher stores most app data locally on your device. This may include settings, workspace layout, dock configuration, theme choices, preferences, backups, and similar launcher data.

Some technical diagnostic information, such as crash reports, may be transmitted securely to Firebase Crashlytics.

If you use backup or restore features, exported backup files may be created by you and stored or shared through Android’s file handling system.

Crash reporting via Firebase Crashlytics is enabled by default and cannot currently be disabled from within the app.

7. Permissions and Device Access

NeoLauncher may request permissions or special access to enable optional features:

  • Contacts permission to support contact search
  • Calendar permission to show event-related features
  • Location permission to provide weather functionality
  • Notification access to show unread badge counts on app icons
  • Accessibility access to enable visual overlay features such as battery ring and edge lighting
  • Device admin access only if you choose to use supported device control actions such as screen lock

Accessibility access is used solely to support visual overlay functionality such as battery ring and edge lighting. It is not used to read personal messages, collect notification content, or monitor private user activity for advertising purposes.

For unread badges, NeoLauncher uses notification access only to determine app-related badge information such as package names and counts. It does not use this feature to sell or profile user data.

If you deny any permission or special access, the related feature may be limited or unavailable, but the app can still function in a reduced mode.

8. Your Choices and Controls

  • You can deny or revoke runtime permissions at any time through Android system settings.
  • You can disable notification access, accessibility access, or device admin access through Android system settings.
  • You can reset launcher settings and data using in-app reset options where available.
  • You can uninstall the app to remove locally stored app data from your device, subject to normal Android behavior.
  • You can choose not to use optional features such as contact search, calendar access, weather, battery ring, edge lighting, or unread badges.

9. Data Security

We take reasonable steps to protect information by limiting collection, keeping most processing on-device, and using established third-party services only when needed to provide app functionality.

However, no method of storage, transmission, or electronic processing can be guaranteed to be completely secure.

10. Children’s Privacy

NeoLauncher is not directed to children, and we do not knowingly collect personal information from children.

11. Changes to This Policy

This Privacy Policy may be updated from time to time. Any updates will be reflected by revising the “Last updated” date at the top of this page.
