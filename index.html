1. Overview

NeoLauncher is an Android launcher app. Most data used by NeoLauncher stays on your device and is processed locally. Some optional features send limited data to third-party services only when needed to provide those features.

No personal data sale No advertising SDKs Local-first launcher data Optional analytics control

2. Information NeoLauncher May Access

Depending on the features you enable, NeoLauncher may access, process, or store:

  • Installed app metadata, such as app names, icons, package names, and component information so the launcher can display and organize your apps.
  • Contacts, if you grant permission, to support contact search and quick actions.
  • Calendar event information, if you grant permission, to show upcoming events and related search results.
  • Approximate or precise location, and saved weather location details, if you grant permission, to provide weather information.
  • Notification data, if you grant notification access, including app package names, unread counts, and the latest notification title/text preview shown on the home screen.
  • Battery level and charging state for Battery Ring and related visual features.
  • Front-camera availability state so visual overlays can hide while the camera is active.
  • User-entered search queries when you use web search, suggestions, or optional external search actions.
  • Media session metadata, such as title, artist, and album art when you use media features.
  • App settings, home-screen layout, dock data, backup data, and imported font files stored locally on your device.
  • Crash diagnostics and limited technical metadata, such as app version and launcher style, if crash reporting is enabled.
  • Anonymous usage analytics, such as feature interactions and general app usage patterns, if analytics sharing is enabled.

3. How We Use Information

NeoLauncher uses this information only to provide launcher functionality and improve app stability. This includes:

  • Displaying and organizing your apps, widgets, home-screen layout, and dock
  • Providing contact search, calendar features, and media features
  • Showing weather information and location names
  • Supporting search, suggestions, and optional external search actions
  • Showing unread badge counts and notification previews on the home screen
  • Rendering Battery Ring and Edge Lighting effects
  • Saving your preferences, backups, and personalization choices
  • Diagnosing crashes and improving reliability
  • Understanding how features are used in aggregate to guide future improvements

4. Third-Party Services and Data Sharing

NeoLauncher does not sell your personal data and does not share personal data for advertising purposes.

When you use relevant optional features, limited data may be sent to:

  • Firebase Crashlytics, if you keep crash reporting enabled
  • Firebase Analytics, if you keep analytics sharing enabled, for anonymous and aggregated usage data only
  • Open-Meteo and api.met.no for weather data
  • Android Geocoder or OpenStreetMap Nominatim for reverse geocoding and place names
  • sunrise-sunset.org for sunrise and sunset times used by weather-related displays
  • Search providers, such as Google, Bing, DuckDuckGo, Yahoo, and Brave, when you use web search or suggestions
  • External apps or services you choose to open, such as voice search, Google Lens, Gemini, phone, SMS, browser, calendar, music, or downloads apps
Contacts, calendar data, notification previews, media information, battery data, and most launcher settings remain on your device unless you explicitly use a feature that hands data to another app or service.

5. Advertising and Analytics

NeoLauncher:

  • Does not show ads
  • Does not include advertising SDKs
  • Does not profile users for marketing purposes
  • Uses Firebase Analytics to collect anonymous, aggregated usage data to improve the app — this can be disabled in Advanced settings

Analytics data collected is anonymous, aggregated, and never linked to your identity. No contacts, location, notification content, or search queries are included in analytics data.

6. Storage and Retention

Most launcher data is stored locally on your device, including settings, layout data, database records, imported font files, and crash-recovery logs.

If you create, export, restore, or share a backup, that backup file is stored or shared only where you choose. Android device backup may also copy supported launcher data according to your device backup settings.

If crash reporting is enabled, crash reports may be sent securely to Firebase Crashlytics. If analytics sharing is enabled, anonymous usage events are collected by Firebase Analytics and retained according to Google's Firebase data retention policies.

7. Permissions and Device Access

NeoLauncher may request or use the following optional permissions or access to enable features:

  • Contacts — for contact search and quick actions
  • Calendar — for event display and search helpers
  • Location — for weather
  • Notification access — for unread badges and home-screen notification previews
  • Accessibility access — for Battery Ring and Edge Lighting overlays and related system-event handling
  • Device admin access — for optional device control actions such as locking the screen

NeoLauncher does not use these permissions for advertising or user profiling.

8. Your Choices and Controls

You can:

  • Grant or deny runtime permissions at any time in Android settings
  • Enable or disable notification access, accessibility access, or device admin access
  • Enable or disable crash reporting in Advanced settings
  • Enable or disable analytics sharing in Advanced settings
  • Reset launcher settings, clear app data, or uninstall the app
  • Choose not to use optional online features

9. Data Security

NeoLauncher minimizes data collection and keeps most processing on-device. Reasonable safeguards are used, but no system can guarantee complete security.

10. Children's Privacy

NeoLauncher is not directed to children and does not knowingly collect personal information from children.
